# Lark Codex အလိုအလျောက်စနစ်

Lark-to-Codex အလိုအလျောက်လုပ်ဆောင်မှုကို လုပ်ဆောင်ရန် ဒေသတွင်းဝဘ် UI ။

## လုပ်ဆောင်ရန်

```bash
python3 automation_web.py
```

ဖွင့်ရန်:

```text
http://127.0.0.1:8787
```

macOS တွင် ဆာဗာလည်ပတ်နေစဉ် `Control + Option + S` ကိုနှိပ်၍ ချက်ချင်းရပ်တန့်ရန် တောင်းဆိုပါ။ UI ရှိ Stop ခလုတ်သည် တူညီသော ဖျက်သိမ်းခြင်းလမ်းကြောင်းကို အသုံးပြုသည်။

## စီးဆင်းမှု

- နံနက်ပိုင်းတွင် သတ်မှတ်ထားသော တက်ရောက်မှုမက်ဆေ့ချ်ကို အစီရင်ခံစာချက်တင်သို့ ပို့သည်။
- အလုပ်ချိန်များသည် သတ်မှတ်ထားသော Lark chat ID များကို သတ်မှတ်ထားသော ကြားကာလတွင် စကင်န်ဖတ်သည်။
- ပြောင်းလဲသွားသော chat အကြောင်းအရာကို သတ်မှတ်ထားသော backend/API သော့ချက်စကားလုံးများနှင့် ကိုက်ညီမှုရှိမရှိ စစ်ဆေးသည်။
- ကိုက်ညီသောမက်ဆေ့ချ်များသည် သတ်မှတ်ထားသော backend project နှင့်အညီ Codex ကို လုပ်ဆောင်သည်။
- Auto-push ကို မူရင်းအတိုင်း ပိတ်ထားသည်။ dry-run စမ်းသပ်ပြီးမှသာ ဖွင့်ပါ။
- ညနေပိုင်းတွင် Codex ဖြင့် အစီရင်ခံစာထုတ်လုပ်ပြီး အစီရင်ခံစာချက်တင်သို့ ပို့သည်။

## ဘေးကင်းရေး မူရင်းများ

- `dry_run` ကို မူရင်းအတိုင်း ဖွင့်ထားသောကြောင့် Lark/Codex/git လုပ်ဆောင်ချက်များကို မှတ်တမ်းတင်ထားသော်လည်း လုပ်ဆောင်ခြင်းမရှိပါ။
- `dry_run` တွင် စကင်န်ဖတ်ခြင်းသည် Lark မက်ဆေ့ချ်များကို ဖတ်နိုင်သေးသည်။ ပို့ခြင်း၊ Codex လုပ်ဆောင်ခြင်းနှင့် git push တို့ကို ပိတ်ထားသည်။
- `auto_push` ကို မူရင်းအတိုင်း ပိတ်ထားသည်။
- ပထမဆုံးစကင်န်ဖတ်ခြင်းကို မက်ဆေ့ချ်ဟောင်းများမှ ဖြစ်ပေါ်လာနိုင်သော အစပျိုးမှုများကို လျှော့ချရန်အတွက် အခြေခံအဖြစ် မှတ်ယူသည်။
- Runtime config ကို `automation_config.json` တွင် သိမ်းဆည်းသည်။
- Runtime state ကို `.automation_state.json` တွင် သိမ်းဆည်းသည်။

## မက်ဆေ့ချ်အရင်းအမြစ်

`read_source` ဖြစ်နိုင်သည်-

- `ui`: ဒေသတွင်း Lark desktop အက်ပ်ကို အသုံးပြုပြီး မြင်နိုင်သော မက်ဆေ့ချ်စာသားကို ကူးယူသည်။ ၎င်းသည် bot မလိုအပ်ဘဲ သင်၏ log-in လုပ်ထားသော Lark အသုံးပြုသူ ဖွင့်နိုင်သော group နှင့် direct chat ID များအတွက် အလုပ်လုပ်သည်။
- `api_dm`: bot အတွက် direct message များအတွက် Lark OpenAPI ကို အသုံးပြုသည်။ bot သည် target chat ကို ဝင်ရောက်ခွင့်ရှိမှသာ ၎င်းကို အသုံးပြုပါ။
- `api`: group-chat API mode ။ bot သည် target group တိုင်း၏ အဖွဲ့ဝင်ဖြစ်မှသာ ၎င်းကို အသုံးပြုပါ။

UI mode သည် Lark AppLink ဖြင့် chat များကို ဖွင့်သည်:

```text
https://applink.larksuite.com/client/chat/open?openChatId=oc_xxx
```

no-bot mode အတွက် `read_source = ui` ကို ထားရှိပြီး group/direct `oc_...` chat ID တိုင်းကို `target_chats` တွင် တစ်ကြောင်းချင်းစီ ထည့်ပါ။

API mode အတွက် ၎င်းတို့ကို UI သို့မဟုတ် environment တွင် သတ်မှတ်ပါ:

```bash
export LARK_APP_ID=cli_xxxxx
export LARK_APP_SECRET=xxxxx
```

Lark အက်ပ်/bot ကို target chat များတွင် ထည့်သွင်းထားရမည်ဖြစ်ပြီး message history permissions ရှိရမည်။

`send_source` သည် ယခုအချိန်အထိ `ui` အဖြစ် ရှိနေသည်၊ ဆိုလိုသည်မှာ နံနက်/ညနေ ပို့စ်များသည် desktop Lark flow ကို ဆက်လက်အသုံးပြုနေဆဲဖြစ်သည်။

API DM mode အတွက် scope ကို ထည့်ပါ-

```text
im:message.p2p_msg:readonly
```

`read_source = api_dm` ဖြစ်သောအခါ `dm_target_id` ကို သင်စကင်န်ဖတ်လိုသော direct message thread သို့မဟုတ် user target သို့ သတ်မှတ်ပါ။ DM-only mode အတွက် group chat list ကို ဗလာထားပါ။ `dm_target_id` ဗလာဖြစ်ပါက scan သည် group chat များကို မထိဘဲ ရှင်းလင်းသောသတိပေးချက်ဖြင့် ရပ်တန့်သွားမည်။
